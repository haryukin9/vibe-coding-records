第2話の最後で「もうひとつの修羅場」として予告した、144件の引っ越しの話です。新しいテーマができて、いざ本番に乗せ替えようとした瞬間に、想定外の景色が広がっていました。

## 第1章 「準備完了」と報告した、その翌日のこと

前回までで、「壊れた朝」から10日ほどかけて、ローカルに新しいテーマと主要コンテンツを揃えるところまでの話を書きました。12月18日にAIに「テーマと主要コンテンツの準備が終わりました」と報告できて、自分の中ではここで一度、勝った気でいました。

あとは本番のサイトに、このローカルの新しいテーマを反映させればいいだけ。
**10秒で終わる作業のはずです。**

実際、その作業自体は10秒で終わりました。
問題は、そのあとブラウザで実績ページを開いた瞬間に起きました。

**実績ページが、ぜんぶ壊れていました。**

動画の枠は表示されない。情報をまとめていた表は崩れている。本文の中に、`<span class="marker_line">` みたいな文字列が、コードのまま画面に出ている。

第1話で書いた「壊れた朝」のときに見た、ショートコードが画面に出ているあの絵面と、ほぼ同じ景色でした。違うのは、**今度は自分が壊した側だ**、ということだけです。

ただ、本当に最悪の発見は、その後にきました。

**実績ページは、当時144件ありました。**

壊れているのは1件や2件じゃない。新しいテーマに切り替えたことで、旧テーマ用に書かれていた装飾やレイアウトのコードが、ぜんぶ「ただの文字列」として吐き出されている。それが144件、横並びで全滅しているわけです。

第2話の最後で「もうひとつの修羅場」と予告した、144件の引っ越し。
始まったのは、ここからでした。

## 第2章 まずは手で直してみた、最初の30分

![画像: 「5分×144件 = 12時間」と紙に書いた手元のメモが、机の上で重く見える](images/png/03-02.png)

最初の発想は、いまから思えば素直すぎました。
**「壊れたコードを、AIに正しい形に直してもらえばいいじゃないか」** と。

仕組みはわかります。新しいテーマには新しいルールでコードを書く必要があって、古いテーマで書かれていたコードがそのまま残っているから、新しいテーマでは「ただの文字列」として読まれてしまう。だったら、古いコードを新しいルールに翻訳すればいい。AIなら、できそうじゃないか。

そこでまず、すでに新しいテーマで正しく表示されている1件を「お手本」として用意しました。Geminiに、こう投げます。

> 「これが正しい形です。これに合わせて、こちらの壊れたコードを直してください」

返ってきます。たしかに、それっぽく直っている。
ローカルのページに貼り直してみると、レイアウトが戻る。動画も再生される。
1件、復活。

5分くらいで終わりました。「これ、行けるな」と思いました。

2件目。同じ手順で投げる。直る。
3件目。直る。

ただ、何件目かで、ふと気づきます。

**これ、144件やるのか？**

仮に1件あたり5分。
144件 × 5分 = 720分 = **12時間**。

これは「途切れずに集中できた場合」の数字です。途中でAIが直しきれない難物が出てきたら、そこに30分溶ける。壊れ方は1パターンじゃないから、テンプレ化もできない。1件ごとに「正しいですか？」と確認する手間も乗ってきます。

「ぶっ通しで2日かければ終わる」と聞こえるかもしれませんが、現実には子どもの面倒も、本業の打ち合わせも、メールの返信もあるわけです。引っ越しに丸2日溶かすことは、現実的に不可能でした。

そこで、また手を止めました。

AIにコードを書いてもらうことは、たしかにできる。でも、**「人間が1件ずつコードを流し込んで、結果を見て、貼り直す」というやり方そのものが、144件というスケールに耐えられない。**

考え方を変える必要がある、と気づいた瞬間でした。

## 第3章 「変換ツールをAIに作らせる」という発想

![図解: AIを「1件分の労働力」として144回呼ぶ図と、AIに「ツール」を作らせて144件を流す図の比較](images/png/03-03.png)

人間が1件ずつやるのが無理なら、「ツールに通す」工程を間に挟めばいいんじゃないか、と思いました。

ひとつ救いがありました。
旧サイトの「壊れる前」のページのコードは、ブラウザのソース表示で取れる、という事実です。表示が崩れていても、もとのHTMLソース自体はちゃんと残っています。

つまり、流れとしてはこうです。

1. 旧サイトのページソースをコピーする
2. 何らかのツールに通す
3. 新テーマ用のフォーマットに整形されたHTMLが返ってくる
4. それを、新サイト側に貼り付ける

**この「2」のツールを、AIに作らせる。** これが次の方針でした。

ここで第2話と同じやり方を取ります。「いきなりコードはいらない。先に何を作るかを決めさせてください」と。

新フォーマットには決まりがあります。

- 動画は新しい埋め込み枠に整形する
- 画像は新しいレイアウトで3枚まで並べる
- 表は白基調のテーブルに作り直す
- 見出しは青のボーダーつき
- 本文中の「マーカー風」の装飾は、新しいコードに置き換える
- 問い合わせボタンには、ページごとに違うパラメータをくっつける(あとで「どの実績から問い合わせが来たか」を追えるように)

これらを箇条書きにまとめて、AIに投げました。「ブラウザだけで動くツールにしたい。サーバーは要らない、HTMLとJavaScriptだけで完結する方式で」と添えて。

最初の動くものは、たぶん15分くらいでポンと返ってきました。そこから何度かテストしては、AIに「ここがうまく拾えていない」「この場合はこうしてほしい」と微調整を繰り返して、合計2時間くらいで完成したと思います。実際には、この期間ずっと並行で別の作業(メディア整理、メタディスクリプションの一括生成、Workspace移行の準備など)も走っていたので、正確な切り分けは難しいんですが、ツール作りそのものに使った時間としては数時間レベルです。

**1件5分でコードをAIに直してもらう作業を144件繰り返すはずだったところを、「2時間でツールを作って、あとは流すだけ」に置き換える。** これが、最初に効いた工夫でした。

画面はシンプルです。左に「ソースの貼り付け欄」、右に「変換後のコード欄」、下に「プレビュー」。旧サイトのページソースを左に貼って、ボタンを押す。新テーマ用のフォーマットに変換されたHTMLが、右に出てくる。下のプレビューで、新サイトでどう見えるかも確認できる。

1件あたりの処理は、コピー → 貼り付け → ボタン → コピー → 新サイトに貼る、で **30秒**。
手作業で1件5分かかっていた作業が、ツールを通すことで1件30秒になりました。**12時間かかるはずだった作業が、1時間ちょっとに縮んだ計算です。**

このときに、自分の中で大事なことに気づいていました。

**「AIにコードを書かせる」と「AIにツールを作らせる」は、まったく違う。**

1件ずつコードを直してもらうやり方では、AIは「1件分の労働力」にしかなりません。144件あれば、144回頼むことになる。
でも、1件用のツールを作ってもらえば、そのツールが、人間の代わりに144件を処理してくれる。

仕組みを作らせるほうが、圧倒的に効く。
これが、「コードを書けない非エンジニアでも、AIに仕組みづくりを頼める」と自分の中で腑に落ちた瞬間でした。

## 第4章 「ローカルで見えてる」では、終わらなかった

![図解: ローカルで見ている時はブラウザが旧本番から画像を取得して表示できているが、本番に切り替えると旧本番にアクセスできなくなり3〜4割の画像がリンク切れになる構造図](images/png/03-04.png)

変換ツールができてから、作業は順調に進んでいました。旧サイトのソースを貼って、変換、コピー、新サイトに貼る。1件30秒で減っていく144件。「勝ちパターンを掴んだな」と思っていました。

何件目かの作業中、何かのエラーで気づきました。

**画像URLの3〜4割が、旧本番(壊れる前のサイト)を見に行ったままだったんです。**

変換ツールはHTMLの構造を新しくしてくれます。テーブル、見出し、ボタンも全部新しい。でも、その中に埋め込まれた `<img src="...">` の中身、つまり画像のURLは、変換ツールが触らないので、旧サイトのまま残っていました。

- 旧サイトの画像URL → 旧本番のサーバーを指している
- ローカル新サイトの画像URL → 自分のPCの中(ローカル環境)を指している

別々の場所を指しています。

ここで疑問が出てきます。じゃあ、なぜローカルで作業中に気づかなかったのか。

理由はシンプルでした。**ローカルのブラウザが、ネット越しに旧本番から画像を取ってきていたからです。**

ローカルの新サイトを開く → でも画像URLは旧本番を指している → ブラウザは素直に旧本番から画像を取得 → 画面では何ごともなく画像が表示される。

ローカルでの見た目には、何の問題もない。

ただ、このまま本番に新サイトを乗せたら、どうなるか。
旧本番のドメインを新サイトに切り替えた瞬間、旧本番のファイルにはもうアクセスできなくなります。一方で、新サイトのHTMLには「画像URL：旧本番」と書かれたまま。

**本番リリースした瞬間、3〜4割の画像が、すべてリンク切れになる。**

ここから、画像URLの一括書き換えに入ります。

AIに相談して、リコメンドされたのは、サイト全体を対象にした「文字列の置換」機能でした。エクセルでいうところの「すべて置換」と同じ発想です。「旧本番のURL」をすべて「ローカルのURL」に置き換える、と指示するだけ。さらに、いきなり実行するのは怖いので、本番に書き換える前に「こう書き換えますよ」というプレビューだけ見せてくれる、安全モードつきのものを使うのがいい、と。

抽出した画像URLは、約900レコード。
プレビュー(ドライラン)の結果、書き換え対象は **522件**。

900と522の差はなんだろう、と少しAIと相談しました。原因は、ゴミ箱に入っていた古い記事、重複URL、すでにローカル化されていたURLなどが含まれていたから。実質「いま生きていて、書き換えが必要なURL」は522件だったわけです。

本番直前です。サイトをまるごとバックアップしてくれる別の機能でローカルのデータを保存して、それから522件の置換を実行しました。

書き換え後、ローカルでもう一度確認。画像はちゃんと表示されている。今度はローカル内のファイルを参照しているので、本番に乗せ替えても画像が消える心配はない。

このときに、もう1つ学びました。

第3章で「AIにツールを作らせる」が効いた、と書きました。でも、**ツールを通したあとに、ツールが解決していない部分が必ず残る。** 変換ツールはHTMLの構造を整えてくれるけど、画像URLには触りません。だから、見た目が完成しているように見えても、中身ができているとは限らない。

**「AIで作った仕組み」と「人間が最後に気づく確認作業」、この両方がないと回らない。**
144件の引っ越しでもうひとつ学んだことでした。

## 第5章 結局、「手作業」は残る

![図解: 「AIで作った仕組み(ツール化)」と「人間がローカルで動かして・目で見て・エラーを拾う(確認)」の二輪が組み合わさって144件が回ったというサイクル図](images/png/03-05.png)

壊れた朝から年末まで、約3週間。新しいテーマを本番に当てて全壊した144件を、ローカルの新サイトに移し終えるまで、結局2日かかりました。

2日というのは、最初に試算した「丸2日 = 12時間」のままに見えるかもしれません。でも、内訳は別物でした。

- 2時間で変換ツールを作る
- 1時間ちょっとで144件を流す
- 残った画像URLは、別の仕組みで一括書き換え(522件)
- 最後に、ローカルの新サイトを目で見て、表示崩れがないか確認

今回から持ち帰った学びは、2つです。

1つ目は、**繰り返し作業に出会ったら、まず「ツール化を頼む」を選択肢に入れる**こと。1件5分かかる作業を144回やるより、2時間かけてツールを作るほうが、合計で得をする。これは、ページの引っ越しに限らず、メルマガでも、画像変換でも、データ整理でも、同じ筋肉で効きます。

2つ目は、**AIで作った仕組みには、必ず「ツールが解決していない部分」が残る**こと。画像URLの問題は、ローカルで見ているうちは何の異常も起きなくて、たぶん何かのエラーが出たことで、ようやく気づけました。事前に「ここに穴が空くはずだ」と予測して防いだわけじゃありません。

AIは指示されたことには答えてくれますが、**指示されていない穴は、エラーや違和感が出てくるまで、人間にも見えません。** だから、AIに任せた仕組みでも、最後はローカルで動かして・目で見て・エラーが出るのを観察する、という時間が必ず要る。今回でいうと、その時間を確保していたから、本番に持っていく前にギリ拾えた、という話だと思っています。

中で動かしているからこそ、エラーや違和感に気づいて、その場で直せる。
1人で会社のサイトリニューアルを動かしていて、いちばん効いていたのは、たぶんこの **「ツール化と確認をセットで回せたこと」** でした。

## 第6章 そして、サイトは無事に公開できた

![図解: 壊れた朝(外注400万)→3週間後に公開(外注ゼロ)のBefore/After](images/png/03-06.png)

ここまでで、ローカル環境に新サイトの中身が一通り揃いました。あとは、新しいテーマを本番ドメインに当てる作業です。

本番に乗せ替えて、ブラウザで開いて、画像が全部表示されている。問い合わせフォームから1通テストで送ってみて、ちゃんとメールが飛んでくる。**ここで、3週間の修羅場が、ようやく終わりました。**

最初の見積もりで「外注なら最低400万円」と言われていたものが、結局、**外注ゼロで済みました**。3週間の修羅場と、自分でやりきった経験だけが残った形です。

「会社のWebサイトを5年放置していた人間が、3週間でAIと一緒に作り直した」というのが、ここまでの2話でわたしが書きたかった話でした。
