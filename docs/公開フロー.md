# 公開フロー

執筆〜note公開までの標準フロー。`/note執筆` スキルが内部で使う。

## 全体像

```
1. 執筆（Markdown）              → articles/<番号>-<slug>.md
2. SVG生成（必要なら）           → images/svg/<slug>-<n>.svg
3. SVG → PNG 変換                → images/png/<slug>-<n>.png
4. スクリーンショット（必要なら）→ images/png/<slug>-<n>.png
5. Markdown → WXR 変換           → wxr/<slug>.xml
6. git commit & push             → GitHub Pages反映（1～2分）
7. note インポート               → 下書きとして取り込み
8. note上で最終確認 → 公開
```

## 各ステップの詳細

### 1. 執筆（Markdown）

- 配置：`articles/<番号>-<slug>.md`
- 番号は2桁（01, 02, 03...）。出版順マップに沿う
- slugは英小文字とハイフン（例：`intro`, `web-rewrite`, `seo-pagespeed`）
- 必ず**front matter**を付ける：

```yaml
---
title: 【ひとりDX担当の記録】記事タイトル
date: 2026-04-29
slug: intro
excerpt: 記事の短い説明（noteのカード表示用）
tags: [vibe-coding, claude-code, dx, 内製化]
---
```

### 2-3. SVG生成 → PNG変換

- SVGは `images/svg/<slug>-<連番>.svg` に保存
  - 例：`intro-01.svg`, `intro-02.svg`
- 変換：
  ```
  cd scripts
  python svg2png.py --all
  ```
- **出力先（デフォルト）**：`J:\マイドライブ\AI_support_Projects\svg-pngs\`（Googleドライブ同期、スマホからも閲覧可）
- 採用フロー：
  1. svg2png.py実行→Googleドライブに保存
  2. Readツールでチャット内プレビュー
  3. ハルさんが適合判定→採用
  4. 採用画像のみ `images/png/<slug>-XX.png` にコピー＆リネーム

### 4. スクリーンショット（公開URLのみ自動）

- 撮りたいURLを `urls.txt` 等に1行1URLで書く
- 実行：
  ```
  cd scripts
  python capture.py --batch urls.txt --full-page
  ```
- **出力先（デフォルト）**：`J:\マイドライブ\AI_support_Projects\playwright-captures\`（Googleドライブ同期、スマホからも閲覧可）
- ファイル名はタイムスタンプ＋URLスラッグで自動命名
- ログイン必要な画面は手動撮影（スマホで撮るかブラウザのスクショ機能）

#### 採用フロー
1. capture.py実行→Googleドライブに保存
2. Readツールでチャット内プレビュー
3. ハルさんが適合判定→採用
4. 採用画像のみ `images/png/<slug>-XX.png` にコピー＆リネーム

### 5. Markdown → WXR 変換

- 単一記事を変換：
  ```
  cd scripts
  python md2wxr.py ../articles/01-intro.md
  ```
- 出力：`wxr/01-intro.xml`
- Markdown内の画像パス（`images/png/foo.png` や相対パス）は**自動でGitHub Pages URLに置換される**
- `--status publish` で即公開ステータスにもできるが、デフォルトは `draft`（下書き）

### 6. git commit & push

```
cd C:\Users\hseki\vibe-coding-records
git add articles/ images/ wxr/
git commit -m "記事追加: <slug>"
git push
```

push後、**1～2分でGitHub Pagesに画像が反映される**。
反映前にnoteインポートすると画像が表示されないので注意。

### 7. note インポート

- ブラウザでnoteにログイン
- 設定 → 記事のインポート / エクスポート → インポート
- WXR形式を選んで `wxr/<slug>.xml` をアップロード
- 取り込み後、下書きに記事が入る

### 8. note上で最終確認 → 公開

確認ポイント：
- 見出し階層が正しいか
- 画像が全部表示されているか
- コードブロックが崩れていないか
- ハッシュタグを追加（`#バイブコーディング` `#ClaudeCode` `#DX` 等）
- アイキャッチを設定
- 公開日時を調整

## 画像配信URLの仕様

- ベースURL：`https://haryukin9.github.io/vibe-coding-records/`
- PNG例：`https://haryukin9.github.io/vibe-coding-records/images/png/intro-01.png`
- 反映までのタイムラグ：push後1～2分
- noteは `http(s)://` で始まる JPEG/PNG/GIF のみ取り込めるので、SVG直貼りは不可
